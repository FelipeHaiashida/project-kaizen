## Codebase Patterns
(Lições reutilizáveis descobertas ao longo das iterações. Leia antes de começar.)

- **Stack fixada:** Next.js 14.2 (App Router) + React 18 + TypeScript. NÃO usar Next 15 / React 19.
- **Tailwind é v3.4** (não v4). shadcn/ui está configurado no estilo CLÁSSICO:
  - `app/globals.css` usa variáveis HSL (`--primary: 262 83% 58%` etc.) com `@tailwind base/components/utilities`. NÃO usar oklch nem `@import "tailwindcss"`.
  - `tailwind.config.ts` mapeia as cores via `hsl(var(--...))`, `darkMode: ["class"]`, plugin `tailwindcss-animate`.
  - Componentes shadcn usam `@radix-ui/*` + `class-variance-authority` + `cn()` de `lib/utils.ts`. NÃO usar `@base-ui/react`.
  - Cor primária da marca = violeta ClickUp `#7C3AED` = `hsl(262 83% 58%)`.
  - Ao rodar `npx shadcn@latest add <comp>`: a CLI mais nova pode gerar código estilo "base-nova"/Tailwind v4. Se isso acontecer, adaptar para o estilo clássico (Radix + HSL) antes de commitar.
- **Prisma é v6** (fixado). Prisma 7 removeu `url`/`directUrl` do schema (exige driver adapters + prisma.config.ts) — por isso ficamos no 6. Generator: `prisma-client-js`. Cliente importado de `@prisma/client`.
- **Variáveis de ambiente:**
  - `.env` (gitignored) → `DATABASE_URL` e `DIRECT_URL`. **O Prisma CLI só lê `.env`, não `.env.local`.**
  - `.env.local` (gitignored) → segredos de app (NextAuth, Supabase, Resend). Lido pelo Next.js.
  - `.env.example` (versionado) → documenta todas as variáveis.
- **Banco JÁ está no ar (Supabase).** Projeto `kaizen` ref `kyrvyqtjlrxgzbrhxjiz` (ca-central-1),
  16 tabelas criadas. `DATABASE_URL` = pooler transaction (6543, `?pgbouncer=true`), `DIRECT_URL` = pooler session (5432).
  - **Usar `npx prisma db push` para sincronizar o schema** (NÃO `migrate dev` — o shadow database falha no pooler do Supabase). Sem histórico de migrations por enquanto; ok para esta fase.
  - Testar conexão avulsa: `node --env-file=.env -e "..."` (não há `dotenv` instalado; o Node 24 tem `--env-file`).
  - Falta `SUPABASE_SERVICE_ROLE_KEY` no `.env.local` (necessária só a partir da S03/Storage).
- **Integração MCP do Supabase aponta para OUTRA conta** (anabeatrizleite235), não a do projeto atual. Portanto NÃO dá para usar as tools MCP (list_tables, advisors, logs, apply_migration) contra este banco — usar Prisma diretamente.
- **Singleton do Prisma:** importar `{ db }` de `lib/db.ts` (NÃO instanciar `new PrismaClient()` avulso).
- **Tipos globais:** importar de `@/types` (reexporta models/enums do Prisma).
- **Convenções:** arquivos em kebab-case, componentes em PascalCase. Rotas autenticadas em `/(app)/`, públicas `/login` `/register` `/invite/[token]`. Mutations via Server Actions. Validação com Zod.
- **npm tem allow-scripts ativo:** postinstall de pacotes (ex.: engines do Prisma) pode ser bloqueado com aviso. `npx prisma generate` funciona mesmo assim; se um engine faltar, rode `npm approve-scripts`.
- **Checks de qualidade:** `npm run typecheck`, `npm run lint`, `npm run build`, `npm run format`. Todos devem passar antes do commit.
- **Preview/screenshot:** o `preview_screenshot` deu timeout neste ambiente mesmo com o server retornando 200. Verificação via `preview_logs` (HTTP 200) + `npm run build` é suficiente.
